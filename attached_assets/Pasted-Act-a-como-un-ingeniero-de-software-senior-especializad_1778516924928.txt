────────────────────────────
ARQUITECTURA
────────────────────────────

Usa una arquitectura modular simple:

1. main.py
   - Punto de entrada de FastAPI
   - Sirve frontend estático
   - Incluye rutas API

2. backend/core/config.py
   - Configuración del sistema
   - Tamaño máximo de archivo
   - Rutas de uploads, reports y checkpoint
   - Extensiones permitidas

3. backend/domain/analysis_result.py
   - Entidad o dataclass AnalysisResult
   - Campos:
     - id
     - filename
     - prediction
     - probability
     - confidence
     - explanation
     - model_version
     - created_at
     - mock_mode

4. backend/services/image_validator.py
   - Validar archivo
   - Validar extensión
   - Validar MIME
   - Validar que Pillow pueda abrir la imagen
   - Validar dimensiones mínimas

5. backend/services/model_service.py
   - Cargar modelo .pt si existe
   - Si no existe, activar modo mock
   - Función predict(image_path)
   - Preprocesamiento consistente:
     - abrir imagen
     - convertir RGB
     - crop o resize controlado a 128x128
     - normalización
   - Si hay modelo real:
     - usar PyTorch
   - Si no hay modelo:
     - devolver predicción simulada pero estable
     - indicar mock_mode=True

6. backend/services/report_service.py
   - Generar PDF con ReportLab
   - Incluir:
     - nombre del archivo
     - fecha
     - resultado
     - probabilidad
     - confianza
     - explicación
     - versión del modelo
     - nota académica: “Este resultado es probabilístico y depende del modelo entrenado.”

7. backend/api/routes.py
   Endpoints:

   GET /
   - Devuelve frontend

   POST /api/analyze
   - Recibe imagen
   - Valida
   - Guarda archivo
   - Ejecuta inferencia
   - Guarda resultado en results.json
   - Devuelve JSON con resultado

   GET /api/report/{analysis_id}
   - Genera o devuelve PDF
   - Descarga archivo PDF

   GET /api/health
   - Devuelve estado del sistema
   - Indica si modelo real está cargado o si está en modo mock

IMPORTANTE:
El modo mock solo sirve para probar la aplicación, no debe presentarse como resultado real del modelo.
