CAMBIOS NECESARIOS

1. Separar claramente los tipos de evidencia en la respuesta y en la interfaz

Actualmente puede confundirse:
- "mensaje oculto encontrado"
- "posible mensaje oculto"
- "sin mensaje oculto"
- "sin evidencia detectable"

Quiero que el sistema use esta lógica textual:

A) Si extraction.payload_found = true y sha256_valid = true:
   Estado: "Mensaje oculto encontrado"
   Tipo de evidencia: "Evidencia directa LSB"
   Explicación:
   "Se encontró un payload compatible con el formato StegoDetect mediante cabecera STEGODETECTv1 y validación SHA-256. La extracción LSB tiene prioridad sobre el resultado ML."

B) Si no hay payload compatible, pero el modelo ML supera el threshold configurado 0.46:
   Estado: "Posible esteganografía detectada"
   Tipo de evidencia: "Evidencia probabilística ML"
   Explicación:
   "No se encontró un payload recuperable compatible con StegoDetect, pero el modelo ML detectó patrones estadísticos asociados a esteganografía. Este resultado es probabilístico y no prueba por sí solo la existencia de un mensaje extraíble."

C) Si no hay payload compatible y el score ML es menor a 30%:
   Estado: "Imagen sin evidencia esteganográfica detectable"
   Tipo de evidencia: "Sin cabecera StegoDetect y baja evidencia ML"
   Explicación:
   "No se encontró payload compatible con el formato StegoDetect y el modelo ML obtuvo un puntaje bajo de evidencia esteganográfica. Según los métodos aplicados por el sistema, la imagen se considera sin evidencia esteganográfica detectable."

D) Si no hay payload compatible y el score ML está entre 30% y el threshold 0.46:
   Estado: "Sin evidencia concluyente"
   Tipo de evidencia: "Baja sospecha ML bajo el umbral configurado"
   Explicación:
   "No se encontró payload compatible con StegoDetect. El modelo ML muestra cierta variación estadística, pero no supera el umbral configurado para clasificar la imagen como sospechosa."

Evitar frases absolutas como:
- "No contiene ningún mensaje oculto"
- "La imagen está 100% limpia"
- "0% de probabilidad de mensaje oculto"

Usar en su lugar:
- "Imagen sin evidencia esteganográfica detectable"
- "No se encontró payload compatible"
- "El resultado ML no superó el umbral configurado"
- "Según los métodos aplicados por StegoDetect"

pero bloquear payloads mayores a 2 MB.

Eso confunde al usuario. Quiero que la interfaz y el backend separen claramente:

- Capacidad teórica de la imagen
- Límite configurado por StegoDetect
- Tamaño del payload seleccionado

Ejemplo:

Capacidad teórica de la imagen: 21.67 MB
Límite configurado por StegoDetect: 2.00 MB
Payload seleccionado: 3.77 MB
Estado: excede el límite configurado del prototipo

El mensaje de error debe decir:

"La imagen tiene capacidad suficiente, pero el payload excede el límite configurado por StegoDetect para evitar sobrecarga de memoria, tiempos de espera elevados o bloqueo del servidor."

Buscar en el backend constantes como:
- MAX_PAYLOAD_SIZE
- MAX_PAYLOAD_BYTES
- MAX_UPLOAD_SIZE
- 2097152
- 2 * 1024 * 1024

No necesariamente aumentes el límite. Primero solo mejora el mensaje y la visualización. Si el límite se mantiene en 2 MB, debe quedar explícito.

8. Mantener la prioridad correcta de decisión

No permitir que el ML sobrescriba la evidencia LSB.

Regla obligatoria:

Si payload_found = true y sha256_valid = true:
    final_decision.status = "payload_found"
    final_decision.title = "Mensaje oculto encontrado"
    primary_metric_label = "Certeza de extracción LSB"
    primary_metric_value = 100

Aunque el ML diga 0%, 20%, 70% o 100%, no debe cambiar la decisión final.

Si no hay payload:
    usar ML solamente como evidencia probabilística.

Si score ML < 30%:
    final_decision.status = "no_stego_evidence"
    final_decision.title = "Imagen sin evidencia esteganográfica detectable"

Si 30% <= score ML < 46%:
    final_decision.status = "inconclusive_low_ml"
    final_decision.title = "Sin evidencia concluyente"

Si score ML >= 46%:
    final_decision.status = "ml_suspicious"
    final_decision.title = "Posible esteganografía detectada"

Agregar en la guía o interfaz una explicación breve:

"StegoDetect diferencia entre evidencia directa y evidencia probabilística. La evidencia directa se obtiene cuando el sistema recupera un payload con cabecera STEGODETECTv1 y hash SHA-256 válido. La evidencia probabilística proviene del modelo ResidualSRNetLite fine-tuned con BOSSBase e imágenes externas, y debe interpretarse como sospecha estadística, no como prueba absoluta."

También agregar:

"Cuando el modelo obtiene un puntaje bajo, por ejemplo menor al 30%, y no existe cabecera StegoDetect, el sistema presenta la imagen como sin evidencia esteganográfica detectable según los métodos aplicados."

También agregar:

"El sistema trabaja preferentemente con PNG. Formatos con compresión con pérdida, como JPG, pueden destruir o alterar los bits menos significativos."
