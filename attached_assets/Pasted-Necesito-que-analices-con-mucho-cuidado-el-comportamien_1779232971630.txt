Problema observado:
El sistema ya funciona para:
- ocultar información,
- extraer payloads StegoDetect,
- validar SHA-256,
- analizar imágenes con ML + LSB.

Pero ahora hay un problema serio de interpretación del modelo:

1. Cuando subo imágenes externas que supuestamente no tienen información oculta, por ejemplo wallpapers JPG o imágenes descargadas de internet, el modelo a veces devuelve 100% y la app dice:
   “Posible mensaje oculto detectado”.

2. Cuando subo una imagen cover directamente de BOSSBase, el modelo devuelve 0%, pero el texto dice:
   “El análisis indica con un 0.0% de probabilidad que la imagen no contiene mensajes ocultos detectables”.

Ese texto está mal redactado y genera desconfianza. Si dice 0%, parece que hay 0% de confianza en que sea limpia, cuando en realidad debería significar:
“Puntaje ML de esteganografía: 0%”.

Contexto técnico:
El modelo SRNet-lite fue entrenado principalmente con imágenes BOSSBase convertidas a PNG, con payloads LSB controlados p005, p010 y p020. El threshold guardado en model_metadata.json es aproximadamente 0.0404. Ese threshold fue validado dentro del mismo dominio experimental, pero no necesariamente en imágenes externas como JPG, wallpapers, imágenes a color muy comprimidas o resoluciones 3840x2400.

Por eso necesito que NO trates el score del modelo como una probabilidad absoluta universal. Debe tratarse como un “puntaje ML dentro del dominio de entrenamiento”.
