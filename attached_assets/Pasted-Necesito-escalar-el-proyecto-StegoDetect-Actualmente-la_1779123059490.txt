- Si la imagen es externa o fue generada por otro algoritmo desconocido, el sistema solo puede:
  a) aplicar el detector ML,

  c) reportar si encontró o no una cabecera válida.

FASE 2 — Formato propio de payload

Implementa una cabecera binaria propia para que el sistema pueda saber qué hay oculto.

La estructura sugerida:

MAGIC = b"STEGODETECTv1"

Header JSON:
{
  "version": "1.0",
  "payload_type": "text" | "pdf" | "image" | "binary",
  "filename": "...",
  "mime_type": "...",
  "payload_size": 12345,
  "sha256": "...",
  "bits_per_channel": 1,
  "channels": ["R", "G", "B"],
  "mode": "sequential",
  "seed": null,
  "created_by": "StegoDetect",
  "algorithm": "LSB-1bit-RGB"
}

Formato final a insertar:

[MAGIC][4 bytes header_length][header_json][payload_bytes]

El extractor debe:
- leer primero el MAGIC
- leer header_length
- leer header_json
- leer payload_size
- validar sha256
- si todo está correcto, devolver el payload
