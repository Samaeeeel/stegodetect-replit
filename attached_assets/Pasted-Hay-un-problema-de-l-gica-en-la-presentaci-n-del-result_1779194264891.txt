Hay un problema de lógica en la presentación del resultado de análisis.

La inserción y extracción LSB funcionan correctamente. Probé una imagen generada por el sistema llamada stego_b1e703ed.png que contiene el mensaje de texto:

Papa

En la sección de análisis LSB, la app detecta correctamente:
- Cabecera StegoDetect detectada
- Payload StegoDetect encontrado
- Tipo: text
- Nombre: message.txt
- SHA-256 válido
- Mensaje extraído: Papa

Pero en la parte superior del resultado sigue apareciendo:

“Sin mensaje oculto”
“Probabilidad de detección 0%”

Eso es incorrecto desde el punto de vista del sistema completo, porque aunque el modelo ML no detecte cambios estadísticos fuertes, el extractor LSB sí encontró y validó un payload real del sistema.

Contexto técnico:
El sistema tiene dos fuentes de evidencia:

1. Modelo ML de estegoanálisis:
   - Es probabilístico.
   - Puede dar baja probabilidad cuando el payload es muy pequeño.
   - No debe ser la única fuente para decidir si hay mensaje oculto.

2. Extracción LSB con cabecera STEGODETECTv1:
   - Si encuentra cabecera válida, payload válido y SHA-256 válido, eso es evidencia directa de que sí existe un mensaje oculto.
   - Esta evidencia debe tener prioridad sobre el resultado ML.

Nueva lógica obligatoria de decisión:

Caso A — payload_found = true y sha256_valid = true:
- Resultado principal:
  “Mensaje oculto encontrado”
- Subtítulo:
  “Payload recuperado mediante extracción LSB del sistema”
- Color/estado:
  alerta positiva o técnica, no “sin mensaje”
- Mostrar:
  - mensaje extraído si es texto
  - botón de descarga si es archivo
  - tipo de payload
  - nombre original
  - tamaño
  - MIME
  - SHA-256 válido
  - algoritmo
  - posiciones usadas
- El resultado ML debe mostrarse aparte como:
  “Resultado del modelo ML: baja probabilidad estadística”
  o
  “El modelo ML no detectó una alteración significativa, posiblemente por tamaño reducido del payload.”

Caso B — payload_found = false pero ml_detection.prediction = stego o ml_score >= threshold:
- Resultado principal:
  “Posible mensaje oculto detectado”
- Subtítulo:
  “El modelo ML detectó patrones compatibles con esteganografía, pero no se encontró una cabecera compatible con el formato StegoDetect.”
- Mostrar:
  - probabilidad ML
  - threshold
  - análisis LSB
  - advertencia: “No se pudo extraer payload porque puede usar otro algoritmo, otra clave o no tener cabecera del sistema.”

Caso C — payload_found = false y ML bajo:
- Resultado principal:
  “Sin evidencia de mensaje oculto”
- Subtítulo:
  “No se encontró payload compatible y el modelo ML no detectó patrones significativos.”
- No decir de forma absoluta que no existe mensaje oculto.
- Usar mejor:
  “Sin evidencia detectable”
  en lugar de:
  “Sin mensaje oculto”
